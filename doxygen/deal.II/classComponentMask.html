Detailed Description

该类代表一个掩码，可用于选择有限元的单个矢量分量（另见 该术语条目 ）。它通常有与有限元的矢量分量一样多的元素，人们可以使用 operator[] 来查询某个特定分量是否被选中。

Note
"掩码 "代表一个具有 true 和 false 项的数据结构，通常用于启用或禁用某个特定向量分量的操作。根据这个定义，禁用的向量组件仍然存在
  • 它们只是没有被触及。因此，当你应用分量掩码来插值一个带有 \(C\) 矢量分量的问题时（仅选择一个例子），描述边界值的输入参数仍然必须提供 \(C\) 分量，即使掩码说我们只想把这些分量的一个子集插值到有限元空间。换句话说，分量掩码不代表reduction操作，它代表selection。 这类对象在很多地方被使用，人们希望将操作限制在某个分量子集上，例如在 DoFTools::make_zero_boundary_values() 或 VectorTools::interpolate_boundary_values(). 中。 这些对象可以手工创建，或者更简单，要求有限元使用代码从某些选定的分量中生成一个分量掩码，例如我们创建一个只表示斯托克斯元速度分量的掩码（见 Handling vector valued problems ）。
// Q2 element for the velocities, Q1 element for the pressure
FESystem<dim> stokes_fe (FE_Q<dim>(2), dim,
FE_Q<dim>(1), 1);
FEValuesExtractors::Scalar pressure(dim);
ComponentMask pressure_mask = stokes_fe.component_mask (pressure);

结果是一个分量掩码，在2d中，它的值是[false, false, true]。同样地，使用

FEValuesExtractors::Vector velocities(0);
ComponentMask velocity_mask = stokes_fe.component_mask (velocities);

在2d中会产生一个 [true, true, false] 的掩码。当然，在3D中，其结果将是 [true, true, true, false] 。

◆ operator[]()

bool ComponentMask::operator[] ( const unsigned int  component_index) const

返回一个特定的组件是否被这个掩码所选择。如果这个掩码代表了选择all components的对象的情况（例如，如果它是用默认的构造函数创建的，或者是从bool类型的空向量转换而来的），那么无论给定的参数是什么，这个函数都返回true。

Parameters
component_index该函数应返回该组件是否被选中的索引。如果这个对象代表一个掩码，其中所有组件总是被选中，那么这里允许任何索引。 否则，给定的索引需要在零和该掩码所代表的组件数量之间。

◆ represents_n_components()

bool ComponentMask::represents_n_components ( const unsigned int  n) const

返回这个分量掩码是否正好代表 n 分量的掩码。如果它被初始化为一个正好有 n entries of type bool 的向量（在这种情况下， n 必须等于size()的结果），或者它被初始化为一个空向量（或者使用默认构造函数），在这种情况下，它可以代表一个有任意数量组件的掩码，并且将总是说一个组件被选中，这就是真的。

◆ n_selected_components()

unsigned int ComponentMask::n_selected_components ( const unsigned int  overall_number_of_components = numbers::invalid_unsigned_int) const

返回被这个掩码选中的组件的数量。 由于空的组件掩码代表每个组件都会返回 true ，这个函数可能不知道组件掩码的真实大小，因此它需要一个参数来表示组件的总数量。 如果该对象已经被初始化为一个非空的掩码（即，如果size()函数返回大于0的东西，或者等价地，如果respresent_the_all_selected_mask()返回false），那么该参数可以被省略，而size()的结果被取走。
