Detailed Description

template<typename T>
struct EnableIfScalar< T >

该类提供了一个本地别名 type ，该别名等于模板参数，但只有在模板参数对应于标量类型（即浮点类型、有符号或无符号整数或复数中的一种）的情况下才是如此。如果模板类型 T 不是标量，那么就没有声明类 EnableIfScalar<T> ，因此也就没有本地别名可用。 该类的目的是如果其中一个参数不是标量数字，则禁用某些模板函数。通过（无意义的）例子，考虑以下函数。

template <typename T>
T multiply (const T t1, const T t2)
{
return t1*t2;
}

这个函数可以用任何两个相同类型的参数来调用 T. 这包括一些参数，这显然没有意义。因此，人们可能想把这个函数限制在只有标量，这可以写成

template <typename T>
typename EnableIfScalar<T>::type
multiply (const T t1, const T t2)
{
return t1*t2;
}

在你调用函数的地方，编译器会从参数中推断出类型 T 。例如，在

multiply(1.234, 2.345);

它将推断出 T 是 double, ，由于 EnableIfScalar<double>::type 等于 double, ，编译器将从上面的模板实例化一个函数double multiply(const double, const double)。另一方面，在这样的背景下

std::vector<char> v1, v2;
multiply(v1, v2);

编译器会推断出 T 是 std::vector<char> ，但由于 EnableIfScalar<std::vector<char>>::type 不存在，编译器不会考虑模板的实例化。这种技术被称为 "替换失败不是错误（SFINAE）"。它确保了模板函数甚至不能被调用，而不是导致后来的错误，即操作 t1*t2 没有被定义（或可能导致一些无意义的结果）。它还允许声明一个函数的重载，如 乘以不同类型的参数，而不会导致编译器产生模糊的调用错误。
