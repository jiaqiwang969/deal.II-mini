Detailed Description

template<int dim, int spacedim = dim>
struct DataOutBase::Patch< dim, spacedim >

描述一个dim空间维度的数据补丁的数据结构。 一个补丁由以下数据组成。

  • 角的::顶点，
  • Patch在每个空间方向的单元数的::n_细分，
  • 附在每个顶点的::数据，以通常的lexicographic排序，
  • 关于::邻居的信息。

有关其内容和目的的更多信息，请参见DataOutBase类的一般文档。 在二维的情况下，下图是一个n_subdivisions=4的例子，因为每个补丁内的（子）单元数等于2dim。

◆ data

template<int dim, int spacedim = dim>
Table<2, float> DataOutBase::Patch< dim, spacedim >::data

数据向量。其格式如下。data(i,.)表示属于i个数据向量的数据。data.n_cols()因此等于输出点的数量；这个数字是（subdivisions+1）^{dim}。data.n_rows()等于数据向量的数量。就目前而言，一个数据向量等于一个标量，即使多个标量后来可能被解释为向量。 在每一列中，data(.,j)是输出点j的数据值，其中j表示deal.II中通常的lexicographic排序。这也是<tt>QIterated类提供的点的顺序，当与QTrapezoid类作为子正交使用时。 由于所有要打印的补丁的数据向量的数量通常是相同的，data.size()应该对所有提供的补丁产生相同的值。例外情况是设置了point_are_available的补丁，其中点的实际坐标被附加到'data'字段中，见point_are_available标志的文档。

◆ points_are_available

template<int dim, int spacedim = dim>
bool DataOutBase::Patch< dim, spacedim >::points_are_available

表示内部补丁点的坐标（假设该补丁应该被进一步细分）是否被附加到 data 表中（ true) 或不附加（ false). 后者是默认的，在这种情况下，该补丁内部点的位置是由补丁顶点的（双，三）线性内插计算的。 这个选项的存在是因为补丁点可能是用Mapping（而不是通过线性插值）来评估的，因此必须存储在Patch结构中。
