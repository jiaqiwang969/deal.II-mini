Detailed Description

template<typename T, typename U>
struct ProductType< T, U >

一个具有本地别名的类，它代表了由类型为 T 和 U. 的两个变量的乘积所产生的类型。换句话说，我们想在这样的代码中推断 product 变量的类型。

T t;
U u;
auto product = t*u;

这个结构的本地别名代表了变量 product 会有的类型。

Where is this useful

这个类的目的主要是表示人们需要用来表示正交点的有限元场的值或梯度的类型。例如，假设你在Vector<float>中存储未知数的值 \(U_j\) ，那么在正交点评估 \(u_h(x_q) = \sum_j U_j \varphi_j(x_q)\) 的结果是需要存储为 \(u_h(x_q)\) 的变量，因为 \(U_j\) 是 float 值， \(\varphi_j(x_q)\) 被计算为 double 值，然后积为 double 值。另一方面，如果你将未知数 \(U_j\) 存储为 std::complex<double> 值，并试图在正交点评估 \(\nabla u_h(x_q) = \sum_j U_j \nabla\varphi_j(x_q)\) ，那么梯度 \(\nabla u_h(x_q)\) 需要存储为 Tensor<1,dim,std::complex<double>> 类型的对象，因为当你用复数乘以 Tensor<1,dim> （用于表示标量有限元的形状函数的梯度的类型）时，你会得到这样的对象。 同样，如果你使用的是矢量值元素（有dim成分），并且 \(U_j\) 被存储为 double 变量，那么 \(u_h(x_q) = \sum_j U_j \varphi_j(x_q)\) 需要有 Tensor<1,dim> 类型（因为形状函数有 Tensor<1,dim> 类型）。最后，如果你将 \(U_j\) 存储为类型为 std::complex<double> 的对象，并且你有一个矢量值的元素，那么梯度 \(\nabla u_h(x_q) = \sum_j U_j \nabla\varphi_j(x_q)\) 将导致类型为 Tensor<2,dim,std::complex<double> >. 的对象。 在所有这些情况下，这个类型是用来识别哪种类型需要用于计算未知数与值、梯度或形状函数的其他属性的乘积的结果。
