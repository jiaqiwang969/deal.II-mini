Detailed Description

template<int spacedim, typename Number = double>
class BoundingBox< spacedim, Number >

一个表示任意尺寸spacedim且边与坐标轴平行的盒子的类，也就是一个区域

\[ [x_0^L, x_0^U] \times ... \times [x_{spacedim-1}^L, x_{spacedim-1}^U], \]

其中 \((x_0^L , ..., x_{spacedim-1}^L) and \) (x_0^U , ..., x_{spacedim-1}^U)表示两个顶点（左下和右上），用于表示盒子。 从几何学上看，一个边界盒就是这样。

  • 1D：一个线段（由其顶点按适当顺序表示
  • 二维：一个长方形（由左下角和右上角的顶点V代表）。
.--------V
| |
V--------.
  • 三维：长方体（在这种情况下，两个顶点V遵循惯例，不属于同一个面）。
.------V
/ /|
.------. |
| | /
| |/
V------.

例如，边界盒在平行分布的网格中很有用，可以对网格的每一部分的所有者进行一般描述。 将BoundingBox<spacedim>的横截面与给定的方向正交，可以得到一个低一维的盒子：BoundingBox<spacedim

  • 1>. 在三维中，BoundingBox<3>横截面的两个坐标可以用两种不同的方式排序。也就是说，如果我们将横截面与y方向正交，我们可以将3D坐标排序为 \((x,z)\) 或 \((z,x)\) 的2D坐标。本类使用第二种约定，对应于坐标的循环排序 \(x \rightarrow y \rightarrow z \rightarrow x \rightarrow ... \) 准确地说，如果我们取一个横截面。 正交于 * 横截面坐标排序为 | |:----------—:|:---------------------------------—:| | x | (y, z) | y | (z, x) | z | (x, y) 这是根据函数 coordinate_to_one_dim_higher 所设定的惯例。

int coordinate_to_one_dim_higher ( const int  locked_coordinate,

这个函数定义了一个惯例，当dim+1维中的一个坐标被锁定为一个给定值时，dim维中的坐标应该如何转换为dim+1维中的坐标。 这个约定是这样的。从锁定的坐标开始，我们连续存储低维的坐标，并在越过维度时进行环绕。这种关系是，在二维中，|锁定在二维中|一维坐标|二维坐标||:---------—:|:----------—:|:----------—:||x0|(a)|(x0, a)||x1|(a)|(a , x1)|，在三维中，|锁定在三维|二维坐标|三维坐标||:----------—|。 -----------—:|:-----------—:| | x0 | (a, b) | (x0, a, b) | | x1 | (a, b) | ( b, x1, a) | | x2 | (a, b) | ( a, b, x2) | 给定一个锁定坐标，这个函数将dim维度的坐标索引映射到dim+1维度的坐标索引。

Parameters
locked_coordinate应该在[0,dim+1]范围内。
coordinate_in_dim应该在[0,dim]范围内。
Returns
在[0, dim+1]范围内的一个坐标索引
