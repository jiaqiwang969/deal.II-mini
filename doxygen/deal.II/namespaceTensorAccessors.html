Detailed Description

这个命名空间是一个在通用张量对象（任意等级）上工作的算法的集合。 在一个单独的命名空间中以通用方式实现这种功能的理由是

  • 以方便代码的重用，从而避免代码的重复。
  • 要有一个定义明确的接口，可以交换低级别的实现。

一个张量对象有一个等级的概念，并允许索引操作符的等级-次数递归应用，例如，如果 t 是一个等级为4的张量对象，下面的访问是有效的。

t[1][2][1][4]

deal.II对诸如 Tensor<rank, dim, Number>和 SymmetricTensor<rank, dim, Number>这样的递归对象有自己的实现。 在这个命名空间中实现的方法和算法，是完全通用的。更确切地说，它可以对嵌套的c-style数组进行操作，或者对具有最小接口的类类型 T 进行操作，该接口提供了一个本地别名 value_type 和一个索引操作符 operator[](unsigned int) ，它返回一个 value_type 的（常量或非常量）引用 。

template <...>
class T
{
using value_type = ...;
value_type & operator[](unsigned int);
const value_type & operator[](unsigned int) const;
};

这个命名空间提供了访问、重新排序和收缩此类对象的基元。

constexpr internal::ReorderedIndexView<index, rank, T> TensorAccessors::reordered_index_view ( T &  t)

为等级为 rank 的张量对象的引用 t 提供一个 "张量视图"，其中索引 index 被移到最后。作为一个例子，考虑一个dim=5空间维度的五阶张量，可以通过5个递归 operator[]() 的调用来访问。

Tensor<5, dim> tensor;
tensor[0][1][2][3][4] = 42.;

索引1（第2个索引，计数从0开始）现在可以通过以下方式移到末尾

auto tensor_view = reordered_index_view<1, 5>(tensor);
tensor_view[0][2][3][4][1] == 42.; // is true

Tensor 类型的用法完全是为了举例说明。这个函数实现的机制可用于相当普遍的张量类型 T. 这个重排设施的目的是能够在两个（或多个）张量的任意索引上收缩。

  • 将指数重新排序，铭记在心，并将其置于张量的末端
  • 使用下面的收缩函数，收缩张量的_最后一个元素。
    Note
    这个函数返回一个由数组下标操作符 operator[](unsigned int) 和描述其返回值的别名 value_type 组成的内部类对象。
    Template Parameters
    index要移到最后的索引。索引从0开始计算，因此有效范围是 \(0\le\text{index}<\text{rank}\) 。
    秩递归对象的秩 t
    T秩 rank. T 的递归对象必须提供一个本地别名 value_type 和一个索引操作符 operator[]() ，它返回一个 value_type 的（常量或非常量）引用.

返回一个数组类型 ArrayType 对象 indices. 所描述的 t 类型的一个子对象的引用（常量或非常量）。 例如。

Tensor<5, dim> tensor;
TableIndices<5> indices (0, 1, 2, 3, 4);
TensorAccessors::extract(tensor, indices) = 42;

这相当于 tensor[0][1][2][3][4] = 42. 。

Template Parameters
T一个等级为 rank. T 的递归对象必须提供一个本地别名 value_type 和一个索引操作符 operator[]() ，它返回一个 value_type 的（常量或非常量）引用 。此外，它的张力等级必须等于或大于 rank.
ArrayType一个类似数组的对象，例如 std::array, 或 TableIndices ，至少存储 rank 索引，可以通过 operator[]() 访问。

这个函数收缩两个递归对象 left 和 right ，并将结果存储在 result. 中，收缩是在两个递归对象的_最后_ no_contr 索引上完成的。

\[ \text{result}_{i_1,..,i_{r1},j_1,..,j_{r2}} = \sum_{k_1,..,k_{\mathrm{no\_contr}}} \mathrm{left}_{i_1,..,i_{r1},k_1,..,k_{\mathrm{no\_contr}}} \mathrm{right}_{j_1,..,j_{r2},k_1,..,k_{\mathrm{no\_contr}}} \]

调用这个函数相当于编写以下低级代码。

for(unsigned int i_0 = 0; i_0 < dim; ++i_0)
...
for(unsigned int i_ = 0; i_ < dim; ++i_)
for(unsigned int j_0 = 0; j_0 < dim; ++j_0)
...
for(unsigned int j_ = 0; j_ < dim; ++j_)
{
result[i_0]..[i_][j_0]..[j_] = 0.;
for(unsigned int k_0 = 0; k_0 < dim; ++k_0)
...
for(unsigned int k_ = 0; k_ < dim; ++k_)
result[i_0]..[i_][j_0]..[j_] +=
left[i_0]..[i_][k_0]..[k_]
right[j_0]..[j_][k_0]..[k_];
}

与r = rank_1 + rank_2

  • 2 no_contr, l = rank_1
  • no_contr, l1 = rank_1, and c = no_contr.
    Note
    类型 T1, T2, 和 T3 必须具有rank rank_1 + rank_2
  • 2 no_contr, rank_1, 或 rank_2，分别。很明显，no_contr必须小于或等于rank_1和rank_2。

三个递归对象的完全收缩。

\[ \sum_{i_1,..,i_{r1},j_1,..,j_{r2}} \text{left}_{i_1,..,i_{r1}} \text{middle}_{i_1,..,i_{r1},j_1,..,j_{r2}} \text{right}_{j_1,..,j_{r2}} \]

调用此函数相当于编写以下低级代码。

T1 result = T1();
for(unsigned int i_0 = 0; i_0 < dim; ++i_0)
...
for(unsigned int i_ = 0; i_ < dim; ++i_)
for(unsigned int j_0 = 0; j_0 < dim; ++j_0)
...
for(unsigned int j_ = 0; j_ < dim; ++j_)
result += left[i_0]..[i_]
middle[i_0]..[i_][j_0]..[j_]
right[j_0]..[j_];
Note
类型 T2, T3, 和 T4 必须分别具有等级rank_1, rank_1 + rank_2, 和 rank_3。 T1 必须是一个标量类型。
