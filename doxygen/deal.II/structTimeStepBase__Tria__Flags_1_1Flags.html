Detailed Description

template<int dim>
struct TimeStepBase_Tria_Flags::Flags< dim >

这个结构是用来告诉TimeStepBase_Tria()类应该如何处理网格。它有一些标志，定义了网格应被重新制作以及何时可以删除的时刻。此外，还有一个变量规定了网格是否应该保留在内存中，或者应该在两次使用之间删除以节省内存。

◆ delete_and_rebuild_tria

template<int dim>
const bool TimeStepBase_Tria_Flags::Flags< dim >::delete_and_rebuild_tria

这个标志决定了 sleep 和 wake_up 函数是否要删除和重建三角结构。 虽然对于小问题来说，这不是必要的，但对于大问题来说，这对于节省内存是必不可少的。 原因是内存中可能有几百个时间层，每个时间层都有自己的三角图，如果每个时间层有很多单元，可能需要大量的内存。在所有时间层上总共有100.000.000个单元的情况并不罕见，这使得这个标志可以理解。

◆ wakeup_level_to_build_grid

template<int dim>
const unsigned int TimeStepBase_Tria_Flags::Flags< dim >::wakeup_level_to_build_grid

这个数字表示 wake_up 函数的参数，它应该在这个参数上重建网格。显然，它应小于或等于传递给时间步长管理对象的 look_ahead 数字；如果它相等，那么网格将在第一次调用 wake_up 函数时重建。如果 delete_and_rebuild_tria 是 false, 这个数字没有意义。
