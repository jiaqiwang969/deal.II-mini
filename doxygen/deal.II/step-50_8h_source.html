1153  * <a name="LaplaceProblemassemble_rhs"></a>
1154  * <h4>LaplaceProblem::assemble_rhs()</h4>
1155  *
1156 
1157  *
1158  * 这个三要素中的最后一个函数为无矩阵方法组装右手边的向量--因为在无矩阵框架中，我们不需要组装矩阵，只需要组装右手边就可以了。我们可以通过从上面的`assemble_system()`函数中提取处理右手边的代码来做到这一点，但是我们决定完全采用无矩阵的方法，也用这种方法进行装配。
1159  *
1160 
1161  *
1162  * 结果是一个类似于 @ref step_37 "step-37" 中 "使用 FEEvaluation::read_dof_values_plain() 来避免解决约束 "一节中的函数。
1163  *
1164 
1165  *
1166  * 这个函数的原因是MatrixFree运算符不考虑非同质的Dirichlet约束，而是将所有的Dirichlet约束视为同质的。为了说明这一点，这里的右手边被组装成残差 @f$r_0 = f-Au_0@f$ ，其中 @f$u_0@f$ 是一个零向量，除了在Dirichlet值中。然后在求解的时候，我们可以看到，解决方案是 @f$u = u_0 + A^{-1}r_0@f$ 。这可以看作是对初始猜测为 @f$u_0@f$ 的线性系统进行的牛顿迭代。下面`solve()`函数中的CG解计算了 @f$A^{-1}r_0@f$ ，调用`constraints.distribution()`（直接在后面）增加了 @f$u_0@f$ 。
1167  *
1168 
1169  *
1170  * 显然，由于我们考虑的是一个零迪里希特边界的问题，我们可以采取类似于 @ref step_37 "step-37" `assemble_rhs()`的方法，但是这个额外的工作允许我们改变问题声明，如果我们选择的话。
1171  *
1172 
1173  *
1174  * 这个函数在积分循环中有两个部分：通过提交梯度的负值将矩阵 @f$A@f$ 的负值应用于 @f$u_0@f$ ，并通过提交值 @f$f@f$ 添加右手边的贡献。我们必须确保使用`read_dof_values_plain()`来评估 @f$u_0@f$ ，因为`read_dof_vaues()`会将所有Dirichlet值设置为0。
1175  *
1176 
1177  *
1178  * 最后，system_rhs向量的类型是 LA::MPI::Vector, ，但MatrixFree类只对 ::LinearAlgebra::distributed::Vector. 起作用，因此我们必须使用MatrixFree功能计算右手边，然后使用`ChangeVectorType`命名空间的函数将其复制到正确的类型。

1685  * qpoint < fe_interface_values.n_quadrature_points;

1697  * copy_data_face.values[1] = copy_data_face.values[0];
1698  * };

1702  * estimated_error_square_per_cell[copy_data.cell_index] += copy_data.value;
1703  *
1704  * for (auto &cdf : copy_data.face_data)
1705  * for (unsigned int j = 0; j < 2; ++j)
1706  * estimated_error_square_per_cell[cdf.cell_indices[j]] += cdf.values[j];

1718  *
