Detailed Description

template<class DerivedIterator, class AccessorType>
class LinearIndexIterator< DerivedIterator, AccessorType >

deal.II中的许多类，如FullMatrix、TransposeTable和SparseMatrix，将它们的数据存储在连续的缓冲区中（当然，这些缓冲区的元素所代表的 解释 可能很复杂）。例如，FullMatrix和TransposeTable分别以行大和列大的顺序存储数据，而对于SparseMatrix，从缓冲区位置到矩阵条目的映射 \(\mathbf{A}(i, j)\) 更为复杂。然而，在任何情况下，元素的连续排列都可以实现随机访问迭代。 LinearIndexIterator提供了为这些类编写迭代器所需的大部分功能。LinearIndexIterator本质上是 boost::iterator_facade 的一个简化版本，它假定 AccessorType 提供了某些成员（下文有记录），这些成员完全描述了迭代器的状态。这个类的预期用途是让容器定义自己的访问器类，然后使用奇怪的重复出现的模板模式（CRTP）技术来定义它们的迭代器。例如，这里是一个使用 LinearIndexIterator 来定义自己的迭代器类的容器。

class Container
{
protected:
// forward declaration for friendship

class Iterator;

class Accessor

public:
// const iterators store a const pointer

Container<T>*>::type;

Accessor(const container_pointer_type container,

protected:

Accessor<Constness>>;

Template Parameters
DerivedIterator如上例所示，具体的迭代器类应该使用该类的CRTP技术：这为迭代器提供了模板式的比较和算术运算符。这是必要的，例如， LinearIndexIterator::operator++() 要返回正确的类型。
AccessorTypeLinearIndexIterator假定 AccessorType 模板参数有以下成员，这些成员完全描述了迭代器的当前状态。
  1. 一个名为 container 的指针指向原始容器（例如，相关的稀疏矩阵）。对于 const 迭代器来说，这应该是一个 const 的指针。
  2. 一个名为 linear_index 的数组索引，用于存储容器存储缓冲区中的当前位置。 linear_index 不需要是一个整数：它可以是一个实现了 operator+=, operator< , 和 operator== 的类类型（可转换为容器的正确索引类型）。例如，可以通过实现 operator+= 和 operator- 的乘法因子来实现strided iterator。
此外， AccessorType 应该声明相关的LinearIndexIterator实例化是一个 friend ，并定义一个 size_type 类型。
Note
TransposeTable使用这个模板来实现其迭代器。
