Detailed Description

template<int dim, int n_rows, int n_columns, typename Number, typename Number2>
struct internal::EvaluatorTensorProduct< evaluate_evenodd, dim, n_rows, n_columns, Number, Number2 >

1d-3d形状函数的内部评估器，使用基函数的张量积形式。 这个类对对称情况下的值、梯度和Hessians也用上述函数处理，实现了一种不同的方法。有可能将每个维度的成本从N^2降低到N^2/2，其中N是一维度的数量（形状矩阵中只有N^2/2个不同的条目，所以这是很合理的）。该方法是基于对输入向量的偶数和奇数部分分别应用算子的想法，因为在正交点上评估的形状函数是对称的。例如，在David A. Kopriva的《实施偏微分方程的频谱方法》一书中介绍了这种方法，Springer, 2009，第3.5.3节（偶数-奇数分解）。尽管书中的实验说该方法在N<20的情况下效率不高，但在循环边界为编译时常量（模板）的情况下，它的效率更高。

n_columns变换矩阵中的列数，对应于通常张量收缩设置中的1d形状函数的数量

inlinestatic

这个函数沿着输入数组中张量数据的给定 direction 应用张量积核，对应于一维条纹的乘法。这个函数允许 in 和 out 数组在n_rows == n_columns的情况下别名，也就是说，在 in 和 out 指向相同地址的地方执行收缩是安全的。对于n_rows != n_columns的情况，只有当 one_line 被设置为真时，输出才是正确的。

Template Parameters
方向被评估的方向
contract_over_rows如果为真，张量收缩对给定的 shape_data 数组中的行求和，否则对列求和
add如果为真，结果被添加到输出向量中，否则计算值将覆盖输出中的内容
]type 决定是否使用形状值（type=0）、形状梯度（type=1）或二阶导数（type=2，类似于type 0，但没有两个额外的0条目）中出现的对称性
one_line如果为真，内核只沿着dim-dimensional tensor中的单个1D条纹应用，而不是像 false 情况中的全部n_rows^dim点。
Parameters
shape_data具有 n_rows 行和 n_columns 列的变换矩阵，以行为主的格式存储
in指向输入数据向量的起点
out指向输出数据向量的起点
