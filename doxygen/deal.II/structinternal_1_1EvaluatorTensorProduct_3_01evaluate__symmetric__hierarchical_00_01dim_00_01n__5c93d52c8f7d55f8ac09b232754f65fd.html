Detailed Description

template<int dim, int n_rows, int n_columns, typename Number, typename Number2>
struct internal::EvaluatorTensorProduct< evaluate_symmetric_hierarchical, dim, n_rows, n_columns, Number, Number2 >

使用基函数的张量积形式的1d-3d形状函数的内部评估器。 这个类实现了一种类似于偶数分解的方法，但具有不同类型的对称性。在这种情况下，我们假设单个形状函数已经显示了在正交点上的对称性，而不是在偶数情况下考虑的完整基础。特别是，我们假设形状函数的排序与Legendre基一样，偶数槽（数值阵列的行）中的形状函数是对称的，奇数槽是点对称的。与偶数分解一样，操作的数量是N^2/2，而不是N^2 FMAs（融合乘加），其中N是一维度数。区别在于输入和输出量的对称方式。

Template Parameters
dim应用该类的空间维度
n_rows变换矩阵中的行数，对应于通常张量收缩设置中的1d形状函数的数量
n_columns变换矩阵中的列数，对应于通常张量收缩设置中的1d形状函数的数量
]Number 用于输入和输出数组的抽象数字类型
Number2用于系数数组的抽象数字类型（默认为与输入/输出数组相同的类型）；必须用Number实现操作符*，并产生Number作为输出，才能成为有效类型。

inlinestatic

这个函数沿着输入数组中张量数据的给定 direction 应用张量积核，对应于一维条纹的乘法。这个函数允许 in 和 out 数组在n_rows == n_columns的情况下别名，也就是说，在 in 和 out 指向相同地址的地方执行收缩是安全的。对于n_rows != n_columns的情况，只有当 one_line 被设置为真时，输出才是正确的。

Template Parameters
方向被评估的方向
contract_over_rows如果为真，张量收缩对给定的 shape_data 数组中的行求和，否则对列求和
add如果为真，结果被添加到输出向量中，否则计算值覆盖输出的内容
]type 决定评估是在 shape_data 的偶数行（type=0）还是奇数行（type=1）中对称，以及在奇数行（type=0）或偶数行（type=1）中偏斜对称
one_line如果为真，内核只沿着dim-dimensional张量中的单个1D条纹应用，而不是像 false 情况中的全部n_rows^dim点。
Parameters
shape_data具有 n_rows 行和 n_columns 列的变换矩阵，以行为主的格式存储
in指向输入数据向量的起点
out指向输出数据向量的起点
