Detailed Description

template<typename T>
struct identity< T >

一个模板类，简单地将其模板参数导出为一个本地别名。这个类，虽然一开始看起来毫无用处，但在以下情况下是有意义的：如果你有一个如下的函数模板。

template <typename T>
void f(T, T);

那么它就不能在 f(1, 3.141) 这样的表达式中被调用，因为模板的类型 T 不能以唯一的方式从参数的类型中推导出来。然而，如果模板被写成

template <typename T>
void f(T, typename identity<T>::type);

那么这个调用就变得有效了： T 的类型不能从函数的第二个参数中推导出来，所以只有第一个参数参与了模板类型解析。 这个特征的背景如下：考虑

template <typename RT, typename A>
void forward_call(RT (*p) (A), A a)
{
p(a);
}
void h (double);
void g()
{
forward_call(&h, 1);
}

这段代码不能编译，因为编译器不能决定模板类型 A should be double （来自作为第一参数给定的函数的签名，因为表达式 1 有该类型。当然，我们希望编译器做的是简单地将 1 to double 投入。我们可以通过编写以下代码来实现这一点。

template <typename RT, typename A>
void forward_call(RT (*p) (A), typename identity<A>::type a)
{
p(a);
}
void h (double);
void g()
{
forward_call(&h, 1);
}
