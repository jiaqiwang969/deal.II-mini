307  // Make the derived class a friend. This seems silly, but is actually
308  // necessary, since derived classes can only access non-public members
309  // through their @p this pointer, but not access these members as member
310  // functions of other objects of the type of this base class (i.e. like
311  // <tt>x.f()</tt>, where @p x is an object of the base class, and @p f one
312  // of it's non-public member functions).
313  //
314  // Now, in the case of the @p SparseBlockVanka class, we would like to take
315  // the address of a function of the base class in order to call it through
316  // the multithreading framework, so the derived class has to be a friend.
