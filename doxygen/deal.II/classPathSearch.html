Detailed Description

支持在路径列表和后缀列表中搜索文件。 为支持的每个文件类别维护一个搜索路径列表。一个文件类别是由一个唯一的字符串定义的。提供的类有：

MESH
各种格式的网格输入文件（见GridIn）
PARAMETER
参数文件（<tt>.prm）

使用add_class()可以很容易地添加额外的文件类别。 使用方法。首先，你为某个文件类构造一个PathSearch对象，比如说网格。然后，你使用find()方法获得一个完整的路径名称，你就可以打开文件了。

#include <deal.II/base/path_search.h>
::PathSearch search("MESH");
std::string full_name = search.find("grid");
std::ifstream in(full_name.c_str());
...

这段代码将首先遍历为文件类MESH设置的列表中的所有路径。如果它成功地打开了一个文件，它将返回istream对象。如果没有，它将尝试追加后缀列表中的第一个后缀，并进行同样的操作。以此类推。如果最后没有找到文件，会抛出一个异常。 如果你想把你的搜索限制在某个网格格式，例如<tt>.inp，那么要么在上面的代码中使用"grid.inp"，要么使用替代的find(const std::string&,const std::string&,const char*) 函数

std::string full_name = search.find("grid", ".inp");

路径列表默认以当前目录（"./"）开始，后面可以选择以标准目录的deal.II开始。使用show()来找出一个给定类的路径列表。路径和后缀可以分别用函数add_path()和add_suffix()来添加。

Note
路径列表中的目录应该总是以尾部的"/"结束，而后缀应该总是以点开始。这些字符不会被自动添加（允许你做一些真正的文件名编辑）。

std::string PathSearch::find ( const std::string &  filename,
const char *  open_mode = "r" 
)

在构造函数指定的类中寻找一个文件，并返回其完整的路径名称（包括可能的后缀）。 文件搜索的工作方式是实际尝试打开该文件。如果 fopen 与提供的 open_mode, 成功，那么文件就被找到了，否则搜索继续进行。

Warning
小心使用 open_mode! 特别是，要非常小心地使用"w"! 如果文件不存在，就无法找到它。如果它确实存在， fopen 函数将把它截断成零长度。
Parameters
filename要找到的文件的基本名称，不包括路径成分和后缀。
open_mode移交给 fopen 函数的模式。
