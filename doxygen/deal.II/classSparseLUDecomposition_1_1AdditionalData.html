◆ extra_off_diagonals

template<typename number>
unsigned int SparseLUDecomposition< number >::AdditionalData::extra_off_diagonals

默认情况下， initialize(matrix, data) 函数创建自己的稀疏度。这种稀疏性具有与 matrix 相同的SparsityPattern，有一些额外的对角线，其数量由 extra_off_diagonals 指定。 用户可以给 use_this_sparsity 一个SparsityPattern。 然后使用这个稀疏度， extra_off_diagonals 参数被忽略。

◆ use_previous_sparsity

template<typename number>
bool SparseLUDecomposition< number >::AdditionalData::use_previous_sparsity

如果这个标志为真，initialize()函数就会使用与之前initialize()调用时相同的稀疏度。 当需要解决几个相同稀疏度的线性问题时，这可能是有用的，例如，在同一个三角形上的几个牛顿迭代步骤。

◆ use_this_sparsity

template<typename number>
const SparsityPattern* SparseLUDecomposition< number >::AdditionalData::use_this_sparsity

当一个SparsityPattern被赋予这个参数时，initialize()函数会调用 reinit(*use_this_sparsity) 导致这个稀疏度被使用。 注意， *use_this_sparsity 的稀疏性结构和传递给initialize函数的矩阵不需要相等。 允许填入，也允许过滤掉矩阵中的一些元素。
